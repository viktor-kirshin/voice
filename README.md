# voiceai — транскрибация диалогов с разделением спикеров

**FastAPI-сервис**, который превращает аудиозапись разговора в размеченный
текст. Помимо обычного **voice-to-text** он:

- расставляет **знаки препинания** и регистр,
- **различает спикеров** в диалоге (диаризация),
- **не цензурирует** речь и помечает **нецензурную брань**,
- автоматически определяет **язык** (включая русский).

## Whisper и pyannote: как они работают вместе

Это две независимые модели, которые решают разные задачи, поэтому в проекте они
запускаются **по отдельности**, а затем результаты **выравниваются по таймкодам**.

1. **Whisper** (на сервере vLLM, через OpenAI-совместимый endpoint) отвечает на
   вопрос *«что и когда сказано»*. Клиент шлёт аудио по HTTP и получает сегменты
   вида `(start, end, text)` — уже с пунктуацией и определённым языком
   (запрашивается `verbose_json` с посегментными таймкодами).

2. **pyannote.audio** (`speaker-diarization-3.1`) отвечает на вопрос
   *«кто и когда говорит»*. Он возвращает интервалы речи с метками спикеров
   (`SPEAKER_00`, `SPEAKER_01`, …), но **не знает текста**.

3. **Выравнивание** (`output.py`): каждому сегменту Whisper присваивается тот
   спикер, чей интервал диаризации перекрывается с сегментом сильнее всего
   (метрика — длительность пересечения таймкодов). Так текст получает автора.

```
audio ──► Whisper ──► сегменты [start, end, text]  ┐
      │                                            ├─► выравнивание по перекрытию ─► размеченный текст
      └──► pyannote ─► интервалы [start, end, spk] ┘
```
Зачем так: тяжёлую модель держим на одной GPU-машине и переиспользуем её как
сервис; сам `voiceai` остаётся лёгким. vLLM даёт батчинг и высокую пропускную
способность, а OpenAI-контракт — стандартный интерфейс. Поднимается одной
командой `docker compose`, подробности — в [docker/README.md](docker/README.md).

```bash
# 1) на GPU-сервере поднять Whisper в vLLM
HF_TOKEN=hf_xxx docker compose -f docker/docker-compose.yml up -d

# 2) указать сервису адрес vLLM — поле base_url в запросе или переменная окружения
export VOICEAI_BASE_URL=http://GPU_HOST:8000/v1
```

Для выравнивания со спикерами сервис запрашивает `verbose_json` с посегментными
таймкодами — vLLM возвращает сегменты `start/end/text`, как и облачный OpenAI.

## Стек

- **Python 3.13** (требуется `>=3.13,<3.14`)
- **uv** — управление окружением и зависимостями (проект уже инициализирован)
- `fastapi` + `uvicorn` — HTTP-сервис и ASGI-сервер
- `python-multipart` — приём загружаемых файлов в `/transcribe`
- `openai` — клиент к OpenAI-совместимому endpoint Whisper (vLLM)
- `pyannote.audio` (>=4.0) — диаризация спикеров (нужен токен HuggingFace)
- `ffmpeg` — декодирование аудио для диаризации (системная зависимость)
- **vLLM в Docker** — обязательный сервис, на нём крутится Whisper
