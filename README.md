# voiceai — транскрибация диалогов с разделением спикеров

**FastAPI-сервис**, который превращает аудиозапись разговора в размеченный
текст. Помимо обычного **voice-to-text** он:

- расставляет **знаки препинания** и регистр,
- **различает спикеров** в диалоге (диаризация),
- **не цензурирует** речь и помечает **нецензурную брань**,
- автоматически определяет **язык** (включая русский).

На входе — загруженный по HTTP аудиофайл (`POST /transcribe`), на выходе —
**JSON** с разметкой сегментов либо готовый текст (`response_format=txt`). Это
**начальный этап** проекта: базовый пайплайн работает, часть возможностей из ТЗ
(эмоции/тональность) пока в планах — см. [Статус реализации](#статус-реализации)
и [Дальнейшие шаги](#дальнейшие-шаги).

## Как решается каждая задача

| Задача | Чем решается |
|---|---|
| **Voice-to-text** (распознавание речи) | OpenAI **Whisper** на сервере **vLLM** (OpenAI-совместимый endpoint) |
| **Знаки препинания** | Whisper расставляет пунктуацию и регистр нативно |
| **Различение спикеров** (диаризация) | **pyannote.audio** (`speaker-diarization-3.1`) |
| **Нецензурная брань** | Whisper не цензурирует текст; пометка по словарю корней |
| **Многоязычность** | Whisper автоопределяет язык (вкл. русский) |
| **Эмоции / тональность** | _запланировано_ (SER на базе `wav2vec2`) |

## Whisper и pyannote: как они работают вместе

Это две независимые модели, которые решают разные задачи, поэтому в проекте они
запускаются **по отдельности**, а затем результаты **выравниваются по таймкодам**.

1. **Whisper** (на сервере vLLM, через OpenAI-совместимый endpoint) отвечает на
   вопрос *«что и когда сказано»*. Клиент шлёт аудио по HTTP и получает сегменты
   вида `(start, end, text)` — уже с пунктуацией и определённым языком
   (запрашивается `verbose_json` с посегментными таймкодами).

2. **pyannote.audio** (`speaker-diarization-3.1`) отвечает на вопрос
   *«кто и когда говорит»*. Он возвращает интервалы речи с метками спикеров
   (`SPEAKER_00`, `SPEAKER_01`, …), но **не знает текста**.

3. **Выравнивание** (`output.py`): каждому сегменту Whisper присваивается тот
   спикер, чей интервал диаризации перекрывается с сегментом сильнее всего
   (метрика — длительность пересечения таймкодов). Так текст получает автора.

```
audio ──► Whisper ──► сегменты [start, end, text]  ┐
      │                                            ├─► выравнивание по перекрытию ─► размеченный текст
      └──► pyannote ─► интервалы [start, end, spk]  ┘
```

pyannote требует токен HuggingFace (модель gated): нужно один раз принять
условия модели на сайте и авторизоваться (см. [Настройка](#настройка)).

## Где работает Whisper

Whisper **не запускается локально**. Модель вынесена на **GPU-сервер в
Docker-контейнер с vLLM**, который поднимает **OpenAI-совместимый endpoint**
(`/v1/audio/transcriptions`). Наш сервис `voiceai` при обработке запроса шлёт
туда аудио по HTTP, а диаризацию (pyannote) и склейку делает у себя.

```
клиент ─► FastAPI voiceai ──────── HTTP, OpenAI API ───────► Docker: vLLM + Whisper (GPU)
          • pyannote (диаризация)  POST /v1/audio/transcriptions  • openai/whisper-large-v3
          • выравнивание + вывод ◄ verbose_json: segments ──────  • endpoint :8000/v1
```

Зачем так: тяжёлую модель держим на одной GPU-машине и переиспользуем её как
сервис; сам `voiceai` остаётся лёгким. vLLM даёт батчинг и высокую пропускную
способность, а OpenAI-контракт — стандартный интерфейс. Поднимается одной
командой `docker compose`, подробности — в [docker/README.md](docker/README.md).

```bash
# 1) на GPU-сервере поднять Whisper в vLLM
HF_TOKEN=hf_xxx docker compose -f docker/docker-compose.yml up -d

# 2) указать сервису адрес vLLM — поле base_url в запросе или переменная окружения
export VOICEAI_BASE_URL=http://GPU_HOST:8000/v1
```

Для выравнивания со спикерами сервис запрашивает `verbose_json` с посегментными
таймкодами — vLLM возвращает сегменты `start/end/text`, как и облачный OpenAI.

## Стек

- **Python 3.13** (требуется `>=3.13,<3.14`)
- **uv** — управление окружением и зависимостями (проект уже инициализирован)
- `fastapi` + `uvicorn` — HTTP-сервис и ASGI-сервер
- `python-multipart` — приём загружаемых файлов в `/transcribe`
- `openai` — клиент к OpenAI-совместимому endpoint Whisper (vLLM)
- `pyannote.audio` (>=4.0) — диаризация спикеров (нужен токен HuggingFace)
- `ffmpeg` — декодирование аудио для диаризации (системная зависимость)
- **vLLM в Docker** — обязательный сервис, на нём крутится Whisper
  (см. [docker/README.md](docker/README.md))

## Установка

```bash
# синхронизировать окружение из uv.lock (создаёт .venv)
uv sync

# системная зависимость для чтения mp3/m4a/... (macOS)
brew install ffmpeg
```

## Настройка

Диаризация pyannote использует gated-модель, поэтому нужен токен HuggingFace.
Один раз примите условия модели `pyannote/speaker-diarization-3.1` на
huggingface.co, затем авторизуйтесь:

```bash
uv run hf auth login
```

Токен хранится на уровне пользователя, поэтому подхватывается автоматически —
передавать `HF_TOKEN` вручную не нужно. (Альтернатива: `export HF_TOKEN=hf_...`.)

## Запуск сервиса

Сначала должен быть поднят vLLM-сервер с Whisper (см.
[docker/README.md](docker/README.md)). Затем запускаем сам API:

```bash
# поднять FastAPI-сервис (по умолчанию 0.0.0.0:8080)
uv run voiceai

# или напрямую через uvicorn с автоперезагрузкой при разработке
uv run uvicorn voiceai.api:app --reload --port 8080
```

Хост/порт настраиваются переменными `VOICEAI_HOST` и `VOICEAI_PORT`.
Интерактивная документация (Swagger UI) — на `http://localhost:8080/docs`.

## Эндпоинты

| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/health` | проверка живости (`{"status":"ok"}`) |
| `POST` | `/transcribe` | загрузить аудио, получить расшифровку |
| `GET` | `/docs` | Swagger UI |

### `POST /transcribe`

Принимает `multipart/form-data`:

| Поле | Описание | По умолчанию |
|---|---|---|
| `file` | аудиофайл с записью разговора (обязательно) | — |
| `model` | id модели Whisper на vLLM-сервере | `openai/whisper-large-v3` |
| `base_url` | URL OpenAI-совместимого endpoint vLLM | `http://localhost:8000/v1` |
| `api_key` | ключ для endpoint (vLLM игнорирует) | `EMPTY` |
| `language` | код языка (`ru`, `en`, ...) или `auto` | `auto` |
| `num_speakers` | ожидаемое число спикеров | `2` |
| `device` | `cpu` / `cuda` / `auto` — устройство диаризации pyannote | `auto` |
| `response_format` | `json` или `txt` | `json` |

`base_url` можно задать и переменной окружения `VOICEAI_BASE_URL`.

Примеры:

```bash
# JSON-ответ с полной разметкой
curl -X POST http://localhost:8080/transcribe \
    -F file=@dialog.m4a \
    -F language=ru \
    -F base_url=http://GPU_HOST:8000/v1

# готовый текст
curl -X POST http://localhost:8080/transcribe \
    -F file=@dialog.wav \
    -F response_format=txt
```

## Формат вывода

**JSON** (`response_format=json`, по умолчанию):

```json
{
  "language": "ru",
  "duration": 8.3,
  "speakers": ["SPEAKER_00", "SPEAKER_01"],
  "segments": [
    {"start": 1.2, "end": 4.85, "start_ts": "00:00:01.20", "end_ts": "00:00:04.85",
     "speaker": "SPEAKER_00", "text": "Привет, как у тебя дела?", "profanity": []},
    {"start": 5.1, "end": 8.3, "start_ts": "00:00:05.10", "end_ts": "00:00:08.30",
     "speaker": "SPEAKER_01", "text": "Отлично, бля!", "profanity": ["бля"]}
  ]
}
```

**Текст** (`response_format=txt`):

```
[00:00:01.20 → 00:00:04.85] SPEAKER_00: Привет, как у тебя дела?
[00:00:05.10 → 00:00:08.30] SPEAKER_01: Отлично, бля!  [мат: бля]
```

Реальный пример текстового вывода — `test.txt` (учебное аудио на английском с
корректным разделением двух спикеров).

## Статус реализации

Реализовано:

- ✅ FastAPI-сервис: `POST /transcribe` (загрузка аудио), `/health`, Swagger UI
- ✅ Voice-to-text через Whisper на vLLM (OpenAI-совместимый endpoint в Docker)
- ✅ Диаризация спикеров через pyannote и привязка текста к спикеру
- ✅ Пометка нецензурной лексики (словарь корней, RU + EN)
- ✅ Вывод в JSON с полной разметкой и в текст (`response_format`)

Пока не реализовано (есть в ТЗ):

- ⏳ Эмоции / тональность голоса (SER)
- ⏳ Автоматическое определение числа спикеров (сейчас параметр `num_speakers`, по умолчанию 2)

## Заметки

- Whisper выполняется только на vLLM-сервере — перед запуском сервиса поднимите
  контейнер (см. [docker/README.md](docker/README.md)).
- vLLM требует Linux + NVIDIA GPU и не работает на macOS/Apple Silicon, поэтому
  сервер держат на отдельной GPU-машине, а сервис ходит к нему по `base_url`.
- Диаризация pyannote выполняется в самом сервисе; на CPU она медленная, на GPU
  (`device=cuda`) — быстрее.
- Эндпоинт `/transcribe` синхронный, FastAPI исполняет его в пуле потоков —
  тяжёлая модель не блокирует event loop, но это не очередь задач: для большой
  нагрузки имеет смысл вынести обработку в фоновые воркеры.
- Первый старт vLLM-сервера скачивает веса Whisper — нужен интернет; дальше веса
  берутся из кэша HuggingFace.
